Here’s the correct way to check audits:

1️⃣ Check Server Audits
SELECT 
    name,
    audit_guid,
    audit_file_path AS file_path,   -- only in SQL Server 2012+
    queue_delay,
    is_state_enabled
FROM sys.server_audits;


If your version does not have audit_file_path, use:

SELECT 
    name,
    audit_guid,
    queue_delay,
    is_state_enabled
FROM sys.server_audits;

2️⃣ Check Database Audit Specifications
SELECT 
    name,
    audit_guid,
    is_state_enabled
FROM sys.database_audit_specifications;


✅ Notes:

sys.server_audits shows the audit definitions (where logs go, enabled/disabled).

sys.database_audit_specifications shows which database objects/actions are being audited.

To see the actual audit log entries, you always use:

SELECT *
FROM sys.fn_get_audit_file('C:\SQLAudit\*.sqlaudit', DEFAULT, DEFAULT);
